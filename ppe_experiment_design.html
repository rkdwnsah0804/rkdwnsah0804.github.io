01
데이터 분석 및 전처리
기본 전처리 — 기존 연구와 동일
  • →Letterbox 리사이즈: 종횡비 유지, 회색(114,114,114) 패딩 적용

추가 전처리 — Person 위주 삭제 (3단계 기준)

2,801개 원본 데이터에서 아래 기준에 따라 Person feature 위주 데이터를 삭제하여 2,000개로 감소

1단계 라벨 분석 이미지별 라벨 검사 → Person 인스턴스만 존재하고 다른 클래스가 없는 이미지를 삭제 대상으로 지정 2단계 삭제 후 검증 희소 클래스(vehicle, Mask 등) 이미지 수가 클래스당 최소 기준 이상인지 확인 기준 미달 시 해당 이미지를 삭제 대상에서 제외 3단계 목표 수량 도달 위 과정 반복 → 최종 2,000개 확보
근거 1 — 불균형 해소

라벨링 분석 결과 Person feature가 유독 많아 전체 클래스 학습 오류 및 예측률 저하 가능성 존재

근거 2 — 학습 환경 제약

과도한 데이터는 시간 소요 및 과부하 한계 존재 → 증강 조합을 위한 적정 규모 조정

02
데이터 분류

클래스당 최소 10~20개 샘플 확보 시 평가 지표의 통계적 신뢰도가 충족된다는 기준에 따라, 10개 클래스 기준 클래스당 최소 15개 이상이 보장되도록 validation 및 test를 각각 150개로 지정

03
데이터 증강

1,700개의 learning data 각각에 대해 4가지 증강 기법 중 1개를 랜덤으로 적용하여 증강 이미지 1,700개 생성. validation/test는 공정한 평가를 위해 원본 유지.

증강 기법 (4가지 중 1개 랜덤)
  • ①Perspective — 원근 변환
  • ②대비(Contrast) 조정
  • ③밝기(Brightness) 조정
  • ④Fog — 안개/분진 환경 재현
최종 증강 구조
원본 1,700장 + 증강 1,700장 ────────── 총 3,400장

⚠ 실험 한계 명시
배치 크기 변경 시 YOLOv8 내부 batch 보정으로 실제 학습률이 함께 변동됨 batch=8 → 실제 lr = 0.01 × (8/64) = 0.00125 batch=16 → 실제 lr = 0.01 × (16/64) = 0.0025 batch=32 → 실제 lr = 0.01 × (32/64) = 0.005 따라서 배치 실험 결과는 배치 크기 자체의 효과와 학습률 변동 효과가 혼재되어 있으며, 이를 완전히 분리하지 못한 한계가 존재함
